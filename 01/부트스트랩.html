부트스트랩 소개와 장단점

부트스트랩(bootstrap) : 반응형 웹을 빠르게 만들어주는 html, css, js 프레임워크이다

하나의 css파일로 모바일,태블릿,데스크탑 다양한 기기에 호환성이 뛰어나다

다양한 기능을 제공하여 사용자가 쉽게 웹사이트를 제작하고 유지보수 할 수 있다

프레임워크(framework)는 재사용 가능한 다양한 요소들이 모여있고, 정해진 구조 및 틀 안에서 확장이 가능한 코드로 구성되어 있다

1-1. 부트스트랩 장점

  • pc, table, mobile 디자인에 적합. 시간절약. 크로스브라우징에 최적화
  • 웹브라우저의 크기에 따라 자동으로 정렬되는 그리드 시스템 제공
  • UI디자인 퀄리티가 높아 많은 기업에서 사용
  • 콘텐츠 제작에 필요한 도구를 빠르게 복사하여 사용가능. 개발자,팀별 및 프로젝트 인수인계 받을때 빠르게 작업가능
  • 많은 자바스크립트 플러그인을 지원. 특히 모바일 중심 프레임워크로 발전함

1-2. 부트스트랩 단점

  • 미리 구축된 스타일이 많아서 커스터마이징이 어려움
  • 미리 정해진 클래스(class)선택자를 확실히 인지하고 있어야함. 디바이스별로 어떻게 화면에서 표현되는지 미리 파악해야함
  • 처음부터 자신이 의도한 디자인이 확실하면 본인이 구축하는것이 좋음. 미리 짜여진 기능을 뜯어고쳐야함.
  • 성능면에서 다른 프레임워크와 큰 차이가 없지만, 페이지 로딩 속도가 상당히 느림
  • 부트스트랩을 사용할 경우 코드의 10%만 사용함. 나머지 90%는 다른 코드들을 함께 로딩함
  • 디자인이 정형화됨. 비슷한 페이지 디자인. 복잡한 사이트 개발 시 사용하지 않는 것이 좋음.
